You may perform the following operation any number of times: </p><ul> <li> Select two integers $$$l$$$ and $$$r$$$ $$$(1 \le l \le r \le |a|)$$$ such that $$$r-l+1 \geq k$$$. </li><li> Then, select an index $$$i$$$ such that $$$l\leq i \leq r$$$ where $$$a_i$$$ is the $$$k$$$-th smallest number out of the subarray $$$[a_l,a_{l+1},\ldots,a_r]$$$. If there are multiple possible $$$i$$$, you may select any. For example, consider $$$a = [1, 2, 2, 1, 3], l = 1, r = 5$$$ and $$$k = 3$$$, the possible candidates of $$$i$$$ are indices $$$2$$$ and $$$3$$$. </li><li> Then, delete $$$a_i$$$ from $$$a$$$, concatenating the remaining parts of the array. </li></ul><p>Determine if it is possible to get an array that is a palindrome$$$^{\text{∗}}$$$ after any number of operations. Note that an empty array is considered a palindrome. </p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$An array $$$b=[b_1,b_2,\ldots,b_m]$$$ is a palindrome if for each $$$1 \leq i \leq m$$$, $$$b_i=b_{m+1-i}$$$.</p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases.
