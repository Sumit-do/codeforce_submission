Alice and Bob will play a game with this array.</p><p>Before the game with Bob starts, Alice can increment any element of the array any number of times, each increment taking one move. One increment consists of a single addition by $$$1$$$ to one index.</p><p>After that, the game starts and Bob and Alice take turns, with <span class="tex-font-style-bf">Bob</span> making the first move. On Bob's turn, he can choose any two indices $$$i$$$ and $$$j$$$ and swap the array elements at these indices (note that Bob may choose $$$i = j$$$, in which case the array remains unchanged after the swap).</p><p>On Alice's turn, if $$$a_1$$$ is even, she finds the largest $$$j \leq |a|$$$ such that $$$a_i$$$ is even for all $$$i \leq j,$$$ then sets $$$a_i := a_i/2$$$ for $$$i \leq j.$$$ Otherwise she sets $$$a_1 := a_1 - 1.$$$ If any element becomes zero, it automatically gets erased from the array (and the other elements are re-indexed accordingly). </p><p>The game ends when the array is empty.</p><p>Alice wants to minimize the number of moves she makes, while Bob wants to maximize the number of moves Alice makes. Note that the number of moves is the sum of the number of initial additions and the number of turns Alice plays after.</p><p>How many moves will Alice make with optimal play?</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. $$$10^5$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, output a single integer — the number of moves Alice will make if both players play
